无障碍指南

确保所有用户都能无障碍使用的设计规范。

对比度要求 (Contrast Requirements)

WCAG AA 最低对比度 4.5:1

所有文字内容必须与背景保持至少 4.5:1 的对比度。大文字（18px 粗体或 24px 以上）可降至 3:1。

主要文字 ✓

base-content on base-100: 通过

次要文字 ✓

content-secondary on base-100: 通过

焦点状态 (Focus States)

所有可交互元素必须有清晰可见的焦点指示器，使用键盘导航时尤为重要。

:focus-visible → outline: 2px solid oklch(60% 0.24 280 / 0.8)
链接焦点

减少动画 (Reduced Motion)

@media (prefers-reduced-motion: reduce)

当用户在系统设置中启用"减少动画"时，以下动画将被禁用：

  • 背景渐变动画 (.bg-celestia-gradient)
  • 浮动光球装饰 (.floating-orb)
  • 消息出现动画 (.chat-message)
  • Toast 通知动画 (.toast-enter, .toast-exit)
  • 页面过渡动画 (.page-enter)
  • 卡片悬停变换 (.card-hover)
  • 所有过渡效果 (transition-duration → 0.01ms)

语义化 HTML (Semantic HTML)

使用正确的 HTML 元素确保屏幕阅读器能正确解析内容结构：

推荐

  • <button> 用于按钮
  • <a> 用于链接
  • <nav> 用于导航
  • <main> 用于主内容
  • <header/footer> 用于页眉/页脚

ARIA 属性

  • aria-label 描述元素
  • aria-hidden 隐藏装饰元素
  • role 定义元素角色
